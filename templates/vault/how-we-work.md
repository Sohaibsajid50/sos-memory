# How We Work

## Memory Order

1. Registry
2. Vault identity
3. Project page
4. Daily note
5. Latest local handoff
6. QMD context
7. Continues bridge handoff, when present

## Health Check

Run `sos health-check --repair` after session saves, project updates, and project bootstraps.

## Durable Records

- Daily notes capture session summaries.
- Project pages capture current state and next steps.
- Decision records capture durable choices and rationale.
- QMD indexes the vault and project sources for retrieval.
